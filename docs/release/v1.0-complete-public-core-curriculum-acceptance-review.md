## Decision

**v1.0 hardens the v0.9 beta path rather than expanding feature scope.** This is a
stabilization / publication-readiness release, not a feature release: per the v0.9
acceptance review (`docs/release/v0.9-capstone-beta-acceptance-review.md` § Hand-Off To
v1.0) and the v1.0 scope boundary (`docs/release/v1.0-scope.md`, issue #249), v1.0 accepts
**no** new features, **Detection patterns**, scenario families, schema entities, or advanced
infrastructure. The work is **hardening only**: every frozen core module runs end-to-end,
the synthetic-data / SQLite / schema invariants hold, the public-facing language and the
v1.1–v1.4 extension boundary are mechanically guarded, and the case library + regulatory
source index cover the frozen core.

The repository remains private until the separate v0.1 publication gate is accepted. v1.0
adds no public-release / "we are now public" language and makes no production-readiness or
certification claim. Both **Fraud detection tracks** — Alpine Crest Private Bank
(private-banking) and NovaBank Digital (digital-banking) — remain first-class throughout the
frozen core, supported by the single **Realistic synthetic data model** and its
**Progressive data views**.

## Hardening Posture

v1.0 is **hardening only**. The frozen core curriculum is the ten module directories that
already exist under `notebooks/` (`00_foundations` through `09_capstone`), matching
`docs/ROADMAP.md` § "v1.0: Complete Public Core Curriculum → Required modules" and
`docs/release/v1.0-scope.md`. v1.1–v1.4 advanced tracks (digital assets / crypto; brokerage
and market-abuse analytics; NLP and communication monitoring; advanced production
infrastructure) stay out of the core per ADR-0004 and are not required for v1.0.

## Verification Commands

CI runs exactly (`.github/workflows/ci.yml`, enforced by `tests/test_ci_quality_gates.py`):

remain out of CI.
