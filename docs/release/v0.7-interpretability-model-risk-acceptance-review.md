## HITL: SHAP Dependency-Cost Decision (slice #187)

Per PRD #50: *"Keep SHAP or heavier dependencies optional unless the team accepts the dependency cost."*

**Decision recorded: keep SHAP optional and outside CI.** SHAP pulls in heavier
dependencies and adds install/runtime cost disproportionate to a single
complementary explanation view. The core curriculum already covers
feature-importance and partial-dependence explanations (slice #180) without it.
`shap` is declared behind an optional `shap` extra in `pyproject.toml` and is NOT
installed by `uv sync --extra dev` or required by CI. PR #197 is open for human
sign-off; this review does not self-merge it.
