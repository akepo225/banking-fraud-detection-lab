## Review Boundaries

- The repository remains private until the v0.1 publication gate is accepted. v1.0 is a
  **hardening release**, not a publication event.
- v1.0 is **hardening only**: no new features, **Detection patterns**, scenario families,
  schema entities, or advanced infrastructure (ADR-0002, ADR-0004, `docs/release/v1.0-scope.md`).
- The project is educational only and unaffiliated with any bank, fintech, regulator,
  vendor, or public case source.
- The project uses synthetic data only, includes no real client data, and does not claim
  to reconstruct real events.
- The project does not provide legal, compliance, audit, investment, regulatory, or
  professional advice.
- CI commands are fixed (AGENTS.md, `docs/quality_gates/v0.1-ci.md`):
  `uv sync --extra dev`, `uv run ruff check .`, and unfiltered `uv run pytest`. v1.0 adds
  no CI commands and filters no tests.

## Out Of Scope (Deferred)

These are deliberately not v1.0 blockers (mirrors `docs/release/v1.0-scope.md` triage):

- Optional-extras (`neo4j`, `shap`) documentation tightening — post-v1.0 follow-up.
- Additional worked examples in the troubleshooting doc — post-v1.0 follow-up.
- v1.1–v1.4 advanced tracks — optional extensions on top of v1.0 (ADR-0004), not required.
- The v0.1 publication gate (go-public) — a separate release gate, not v1.0.

## Final Human Review Items

- Confirm v1.0 is framed as hardening only, not a publication event.
- Confirm the two **Fraud detection tracks** (Alpine Crest Private Bank, NovaBank Digital)
  are described accurately and do not imply real-bank affiliation.
- Confirm the frozen 10-module core list (`00_foundations` … `09_capstone`) and the
  v1.1–v1.4 extension boundary match the maintainer's intent.
- Confirm v1.0 adds no new **Detection patterns**, schema entities, or infrastructure.
- Decide whether to keep the repo private for additional polish before the v0.1
  publication gate.

## How to validate v1.0 locally

```bash
uv sync --extra dev
uv run ruff check .
uv run pytest
```

These are exactly the commands CI runs (see
[`docs/quality_gates/v0.1-ci.md`](../quality_gates/v0.1-ci.md)). `uv run pytest` is
unfiltered and includes the v1.0 cross-version regression gate
(`tests/test_v10_acceptance_review.py`) and the data/schema gate
(`tests/test_v10_data_schema_gate.py`).
