# PRD: Banking Fraud Detection Lab v1.3

Status: draft for human review.

## Problem Statement

The lab's core curriculum and advanced structured-data extensions still leave one
important area mostly uncovered: unstructured communication. Fraud investigations
often involve secure messages, support chats, relationship notes, complaint text,
call summaries, and case narratives. Learners need to understand how NLP can help
without replacing human review or introducing privacy risks.

## Solution

Add an optional NLP and communication-monitoring module using synthetic text only.
v1.3 should introduce conversations, messages, channels, participants, complaint
records, relationship notes, communication-derived signals, NLP features,
classification examples, summarization support, and governance controls.

## User Stories

1. As a learner, I want synthetic communications linked to **Client**, **User**, **Partner**, **Banking relationship**, alerts, and cases, so that text is connected to structured data.
2. As a learner, I want metadata-only views before full text, so that **Progressive data views** still apply.
3. As a learner, I want scam, coercion, impersonation, distress, suspicious trading intent, and social-engineering examples, so that NLP features map to **Detection patterns**.
4. As a learner, I want keyword, embedding, and classifier examples, so that methods can be compared.
5. As a learner, I want generated investigation-summary support, so that I can see usefulness and limits.
6. As a learner, I want human-review checkpoints, so that NLP output is not treated as an automatic case decision.
7. As a learner, I want false-positive examples from normal urgent client communication, so that text signals are contextual.
8. As a learner, I want governance notes for privacy, retention, bias, explainability, and sensitive data handling, so that text analytics are treated carefully.
9. As a maintainer, I want synthetic-only generation checks, so that no real personal data is introduced.
10. As a reviewer, I want v1.3 to remain optional, so that text processing is not required for v1.0 completion.

## Implementation Decisions

- Add v1.3 as an optional advanced module outside the v1.0 core path.
- Use only synthetic communication content.
- Add entities for conversation, message, channel, participant, interaction note, complaint, communication signal, and summary.
- Link communications to existing structured entities and **Detection patterns**.
- Add **Progressive data views** for metadata, redacted text, full synthetic text, features, and case-support outputs.
- Treat summaries as assistance for review, not final decisions.
- Add governance material focused on educational risk awareness.

## Testing Decisions

- Test deterministic synthetic text generation and scenario coverage.
- Test joins between communications, transactions, alerts, cases, **Client**, **User**, and **Banking relationship** records.
- Test metadata-only and text views.
- Test NLP feature extraction on fixed synthetic examples.
- Test that generated summaries reference traceable source snippets or records.
- Regression-test v1.0, v1.1, and v1.2 outputs where relevant.
- Pass the full v1.0 test suite and all v1.0 core notebook smoke tests without
  requiring changes to the v1.0 learner path.

## Out Of Scope

- Real email, chat, call, or case-note ingestion.
- Production communication surveillance.
- Legal discovery workflows.
- Fully automated case closure from NLP.
- Real personally identifiable data.
- Making NLP a required core path.

## Cross-Version Links

- v1.3 depends on v1.0 alert, case, governance, and monitoring foundations.
- v1.3 may reuse v1.1 scam and digital-asset scenarios.
- v1.3 may reuse v1.2 brokerage-intent scenarios.
- v1.4 can optionally serve NLP models and monitor communication drift.

## Self-Review Checklist

- The PRD uses synthetic text only.
- The PRD links NLP features to **Detection patterns** and structured evidence.
- The PRD keeps human review central.
- The PRD treats summaries as support, not decisions.
- The PRD keeps NLP optional and protects v1.0 stability.
