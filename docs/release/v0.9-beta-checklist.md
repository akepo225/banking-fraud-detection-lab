# v0.9 Beta Readiness Checklist

Status: accepted for v0.9 beta with bounded v1.0 follow-ups.

This checklist converts the v0.9 PRD into reviewable beta-readiness evidence. It
separates **release-blocking** gates from **polish** items so a maintainer can
decide whether v0.9 is beta-ready. It is an internal/maintainer artifact; it is
**not** a publication announcement. The repository remains private until the
separate v0.1 publication gate is accepted.

## Review Boundaries

- The repository remains private until the v0.1 publication gate is accepted.
  v0.9 is a **beta review point**, not a shipped release.
- The project is educational only and unaffiliated with any bank, fintech,
  regulator, vendor, or public case source.
- The project uses synthetic data only, includes no real client data, and does
  not claim to reconstruct real events.
- The project does not provide legal, compliance, audit, investment,
  regulatory, or professional advice.
- CI commands are fixed (AGENTS.md, `docs/quality_gates/v0.1-ci.md`):
  `uv sync --extra dev`, `uv run ruff check .`, and unfiltered `uv run pytest`.
  The beta checklist references these gates; it does not redefine them.

## Final Beta Decision

v0.9 is accepted for beta on the evidence above. The acceptance is limited to
the v0.9 integration capstone and beta-readiness artifacts: it does not declare
v1.0 complete, it does not change the private pre-publication posture, and it
does not add new **Detection patterns**, scenario families, schema entities, or
advanced infrastructure.

Remaining work after this point belongs to v1.0 hardening or the separate v0.1
publication gate. Polish items below are intentionally not release-blocking.

## Polish (Not Release-Blocking)

- Cross-doc link polish between `README.md`, the notebook guide, the SQL guide,
  and `docs/capstone/` (added in slice #230; re-checked in #232).
- Tightening of optional-extras handling (`neo4j`, `shap`) documentation, if a
  learner reports confusion.
- Additional worked examples in the troubleshooting doc beyond the common
  failures covered below.

## Final Human Review Items

- Confirm the v0.9 beta is framed as a review point, not a shipped release.
- Confirm the two **Fraud detection tracks** (Alpine Crest Private Bank,
  NovaBank Digital) are described accurately and do not imply real-bank
  affiliation.
- Confirm the rubric (`docs/capstone/rubric.md`) grades the capstone against the
  intended learning objectives.
- Confirm source-pack, regulatory-note, and capstone-brief framing avoids legal
  advice and real-event reconstruction claims.
- Review the public issue templates and expanded contributor guidance (slice
  #233, HITL): decide whether public templates land now or defer to v1.0 (the
  PRD orders them after the publication-boundary review).
- Decide whether to keep the repo private for additional polish before the
  v0.1 publication gate.

## How to validate the beta locally

```bash
uv sync --extra dev
uv run ruff check .
uv run pytest
```

These are exactly the commands CI runs (see
[`docs/quality_gates/v0.1-ci.md`](../quality_gates/v0.1-ci.md)). To exercise the
capstone substrate manually:

```bash
uv run python -m banking_fraud_lab.capstone --track both --seed 42 --scale tiny --learner-facing --output data/capstone
```
