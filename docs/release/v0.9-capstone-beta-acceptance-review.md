## Decision

**Conservative Hold pending final human beta decision.** The v0.9 child issues
(#225–#233) are the delivery mechanism; their PRs (#236–#247) carry the
implementation. This review (#234) is the final v0.9 sign-off artifact and is
**not self-merged** — it opens for human sign-off, exactly as the v0.1
publication gate (#44) and the v0.8 acceptance review (#210) do.

v0.9 introduces the v0.9 integration capstone (`09_capstone`) and the public
beta review layer. Learners can walk a single end-to-end path from scenario
brief through synthetic data generation, SQL feature extraction, the track
feature libraries, a fitted score, alert review, capacity-aware threshold
selection, per-alert explanation, graph/case evidence, production monitoring,
and a governance memo. Both **Fraud detection tracks** (Alpine Crest Private
Bank, NovaBank Digital) are represented with distinct **Detection patterns**.
The beta layer adds a release-blocking checklist, clean-setup validation,
troubleshooting, terminology guardrails, and (HITL) public issue templates +
contributor guidance.

The repository remains private until the separate v0.1 publication gate is
accepted; v0.9 is a **beta review point, not a shipped release**. It adds no
public-release language and makes no production-readiness or certification
claim.

## Exit Criteria (PRD / ROADMAP)

- ✅ **Exit criterion 1** ("Learners can walk the integrated capstone path for
  both tracks"): the capstone scoring notebooks (both tracks) and the synthesis
  notebook execute end-to-end via nbconvert on capstone tiny data; the capstone
  SQL examples run against the loaded capstone dataset; the dataset is
  deterministic and isolates protected answer keys. Enforced by
  `tests/test_capstone_scoring_notebook.py`, `test_capstone_synthesis_notebook.py`,
  `test_capstone_sql.py`, `test_capstone_datasets.py`, and the new regression
  gate in `tests/test_v09_acceptance_review.py`.
- ✅ **Exit criterion 2** ("Maintainers can decide beta-readiness against the
  PRD exit criteria, with terminology and reproducibility guardrails"): the
  beta checklist maps release-blockers to PRD exit criteria / quality gates and
  references the fixed CI commands; the clean-setup test proves reproducibility;
  the terminology guardrails lock the v0.9 public-facing surface; the issue
  templates + contributor guidance (HITL) structure post-beta feedback. Enforced
  by `tests/test_v09_clean_setup_commands.py`, `test_publication_docs.py`, and
  `test_issue_templates.py`.
