## ROADMAP v0.5 Exit Criteria Review

The three ROADMAP exit criteria for v0.5 are each satisfied and test-guarded.

### Exit criterion: every core module links to at least one public case or source pack

Pass. Each core module (`notebooks/00_foundations/` through
`notebooks/05_digital_session_and_payment_fraud/`) carries a "Case library and
regulatory context" section in its README (PR #134) that links to at least one
case source pack or regulatory source note plus both indexes. The reverse
direction is now enforced by
`test_every_core_module_has_an_inbound_case_or_regulatory_link`, which asserts
every core module has at least one inbound link from the case library or the
regulatory index. Evidence: all six modules resolve; `notebooks/00_foundations/`
is covered by regulatory notes (`swiss-amla-amlo-finma`, `model-risk-governance`).

### Exit criterion: case documents clearly avoid reconstructing real events

Pass. The case template (`docs/cases/TEMPLATE.md`) and source-quality rubric
(`docs/cases/source_quality_rubric.md`) from issue #118 / PR #126 establish the
non-reconstruction and non-affiliation disciplines, and they are enforced by
`test_case_source_packs_do_not_claim_reconstruction`,
`test_case_source_packs_do_not_include_direct_quote_blocks`, and
`test_case_source_packs_avoid_imperative_compliance_wording`. Synthetic case
material is inspired by public cases but never claims to reconstruct real
events, real clients, or real accounts.

### Exit criterion: regulatory notes avoid legal advice and use official-source links

Pass. Each regulatory source note carries a HITL marker, an "educational use
only" disclaimer, and official-source links on the allow-listed domains
(`fedlex.admin.ch`, `finma.ch`, `psr.org.uk`, `fatf-gafi.org`,
`federalreserve.gov`) over HTTPS only. The learning-use taxonomy (issue #122 /
PR #128) reorganizes the regulatory index by learning use while preserving these
boundaries. Enforced by `test_regulatory_source_index.py` (banned-imperative
patterns, official-domain allow-list, and `learning_use` metadata coverage).

## Domain Boundaries

This review preserves the repo vocabulary and source-boundary language.

- Glossary terms are used exactly: **Client** is the legal customer, **User**
  is the digital login identity, and **Banking relationship** is the
  Swiss-bank-style relationship container.
- Synthetic institutions only: **Alpine Crest Private Bank** for private-banking
  scenarios and **NovaBank Digital** for digital-banking scenarios. No real-bank
  affiliation.
- No real client, account, or transaction data; no real-client-data claims; no
  reconstruction of real events; no legal advice, compliance advice, or
  job-preparation framing.
