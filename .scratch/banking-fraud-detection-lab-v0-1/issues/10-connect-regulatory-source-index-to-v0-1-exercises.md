# Connect Regulatory Source Index To v0.1 Exercises

Status: ready-for-human

GitHub: https://github.com/akepo225/banking-fraud-detection-lab/issues/11

## Parent

GitHub issue #1: https://github.com/akepo225/banking-fraud-detection-lab/issues/1

## What to build

Create the v0.1 regulatory source index and connect it to analytics, alert handling, documentation, explainability, controls, and governance exercises.

This is HITL because regulatory framing must be reviewed to stay educational and avoid legal advice.

## Implementation order

Start after the foundations and governance paths exist, so each regulatory note can link to a real exercise.

## What needs to be implemented first

- Define the regulatory source note template and validation rules.
- Add the educational-only disclaimer to the template.
- Add official-source links and original learning implications.
- Link each source note to at least one v0.1 notebook or exercise.

## Acceptance criteria

- [ ] Regulatory source index includes Swiss AMLA, AMLO, FINMA, APP scam or payment guidance from EU or UK where relevant, FATF typologies, and model-risk or governance references.
- [ ] Source notes link to official sources and quote only short excerpts when necessary.
- [ ] Source notes summarize learning implications in original wording.
- [ ] Source notes connect regulatory material to v0.1 notebooks or exercises.
- [ ] Validation checks verify official links, educational-only disclaimer, learning implications, and no imperative compliance instruction wording.
- [ ] Documentation states that regulatory notes are educational and not legal or compliance advice.
