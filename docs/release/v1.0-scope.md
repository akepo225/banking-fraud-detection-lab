## Decision

**This slice is the scope + triage gate for every other v1.0 slice.** v1.0 is a
stabilization / publication-readiness release, **not** a feature release: per the
v0.9 acceptance review (`docs/release/v0.9-capstone-beta-acceptance-review.md`
§ Hand-Off To v1.0) and the v1.0 PRD Cross-Version Links, *v1.0 hardens the beta
path rather than expanding feature scope* and accepts **no** new features,
**Detection patterns**, scenario families, schema entities, or advanced
infrastructure.

This document is recorded under parent planning issue **#53** (PRD: Banking
Fraud Detection Lab v1.0) and aligns with the v1.0 PRD at
`docs/prds/v1.0-complete-public-core-curriculum.md`. It produces no code, test,
schema, or generator changes; it is a scoping + triage decision recorded in this
single markdown doc.

The repository remains private until the separate v0.1 publication gate is
accepted; v1.0 adds no public-release / "we are now public" language and makes no
production-readiness or certification claim.

## v1.1–v1.4 Stay Out of Core

The v1.1–v1.4 advanced tracks are formally out of the v1.0 core curriculum. This
follows ADR-0004
(`docs/adr/0004-treat-v1-1-through-v1-4-as-optional-advanced-extensions.md`) and
the `docs/ROADMAP.md` § "Post-1.0 Advanced Tracks" section. The following content
must not enter the v1.0 core:

- **v1.1 — digital assets and crypto fraud** (fiat on/off-ramp risk, wallet
  withdrawal risk, destination-risk scoring, crypto regulatory source notes).
- **v1.2 — brokerage and market-abuse analytics** (instruments, orders,
  executions, unauthorized trading patterns, market-abuse indicators).
- **v1.3 — NLP and communication monitoring** (synthetic communications,
  insider-risk / collusion indicators, case-note summarization).
- **v1.4 — advanced production infrastructure** (relational-database deployment
  notes, Neo4j graph export, Kafka / stream-processing labs, dashboard or API
  deployment).

These tracks may later be built as optional advanced extensions on top of the
v1.0 core; they are not required for v1.0 and must preserve the v1.0 learner path
per ADR-0004. This section is the human-readable decision behind the mechanical
boundary enforcement performed by downstream slice **#253** (publication-language
and extension-boundary guardrails), which adds an automated check that v1.1–v1.4
content has not leaked into the v1.0 core docs.

## Publication Boundary

v1.0 is **hardening only**. The go-public switch is not part of v1.0; it remains
the separate **v0.1 publication gate**
(`docs/release/v0.1-publication-checklist.md`, status: ready for human review).
Accepting v1.0 does not publish the repository.

v1.0 adds **no** public-release / "we are now public" language and makes no
production-readiness, certification, or job-market claim. This mirrors the
framing every existing release doc already carries: the repository remains
private until the v0.1 publication gate is accepted.
